🚀 2. Activación de Servicios Críticos
El error en image_847900.png indica que el servicio está apagado. Vamos a forzar su inicio y habilitarlo
para que persista tras los reinicios:

Bash
# Iniciar y habilitar el demonio de red
sudo systemctl enable systemd-networkd
sudo systemctl start systemd-networkd

# Verificar que el servicio dbus-org.freedesktop.network1 esté disponible
sudo systemctl unmask systemd-networkd
📈 3. Aplicación de la Configuración
Una vez que el servicio esté corriendo, aplica los cambios. Si estás por SSH, usa try para evitar quedar 
fuera del servidor si hay un error:

Bash
# Probar la configuración (tienes 120s para confirmar)
sudo netplan try

# Si todo es correcto, aplica definitivamente
sudo netplan apply

maestro (Bash) diseñado para instalar y optimizar toda la pila tecnológica de transmisión, desarrollo y red en un solo paso.
Este script prepara la instancia de Ubuntu 26.04 para manejar codificación en tiempo real, contenedores, y desarrollo backend 
de alto rendimiento.
Ejecuta el siguiente bloque en tu terminal con privilegios de superusuario (sudo):

Bash
#!/bin/bash
# Despliegue de Infraestructura de Streaming y Desarrollo - Ubuntu 26.04 LTS

echo "[1/6] Actualizando el sistema base..."
sudo apt update && sudo apt upgrade -y

echo "[2/6] Instalando utilidades de sistema, red y agentes de Proxmox..."
sudo apt install -y curl wget git htop bmon nload sysstat \
    qemu-guest-agent spice-vdagent build-essential libssl-dev sqlite3

echo "[3/6] Instalando OBS Studio y FFmpeg (Codificación de Video)..."
sudo add-apt-repository ppa:obsproject/obs-studio -y
sudo apt update
sudo apt install -y obs-studio ffmpeg vlc

echo "[4/6] Preparando entorno de desarrollo (Node.js y Go)..."
# Instalación de Node.js (LTS)
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt install -y nodejs
# Instalación de Go (Golang)
sudo snap install go --classic

echo "[5/6] Instalando Docker (Infraestructura para Datarhei/Restreamer)..."
sudo apt install -y docker.io docker-compose
sudo systemctl enable --now docker

echo "[6/6] Aplicando reglas de Firewall (UFW) para Streaming..."
sudo ufw allow 22/tcp      # SSH Management
sudo ufw allow 5002/udp    # Ingreso SRT
sudo ufw allow 1935/tcp    # Ingreso/Egreso RTMP
sudo ufw allow 8080/tcp    # API/Web UI (Restreamer/Nimble)
sudo ufw allow 80/tcp      # HTTP
sudo ufw allow 443/tcp     # HTTPS
# sudo ufw enable          # Descomentar tras confirmar que no se perderá el acceso SSH

echo "Despliegue finalizado. Por favor, reinicie la máquina virtual para que los agentes de VirtIO y el nuevo kernel se sincronicen correctamente."
Nota de Seguridad: He dejado el comando sudo ufw enable comentado por precaución.
